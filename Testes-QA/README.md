# Testes QA

## Propósito

Esta pasta contém a documentação de testes de qualidade (QA) realizados no sistema. Aqui estão organizados os bugs identificados durante os testes, cada um com sua descrição detalhada, passos para reprodução, evidências visuais e comportamento esperado.

Conforme o sistema evolui e novas funcionalidades são adicionadas ou atualizadas, novos bugs podem ser identificados e documentados nesta pasta.

## Estrutura de Organização

Os bugs são organizados em subpastas numeradas sequencialmente (Bug 1, Bug 2, Bug 3, etc.), facilitando a rastreabilidade e organização cronológica dos testes realizados.

## Conteúdo de Cada Bug

Cada pasta de bug contém:

- **README.md** - Documentação completa do bug com:
  - Descrição da funcionalidade testada
  - Passos detalhados para reprodução
  - Comportamento esperado
  - Observações relevantes
- **Evidências** - Imagens (formato JPEG) que comprovam a ocorrência do bug

## Como Usar

Para visualizar os detalhes de um bug específico, navegue até a pasta correspondente e abra o arquivo `README.md`. As evidências visuais estão incluídas diretamente na documentação.

---

**Nota:** Esta documentação segue o padrão de escrita BDD (Behavior-Driven Development) para facilitar a compreensão e reprodução dos bugs identificados.
